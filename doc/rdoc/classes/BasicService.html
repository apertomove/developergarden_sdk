build_security_header_common(doc)

Regardless to whether it is a login, getTokens or a regular service call there always has to be a security header. This method provides the structure which is in common to all these method calls.

The header of the given document will be enhanced so there is no need to process the returning value.

Parameters

doc:Request XmlMason document.

build_service_header(doc, security_token)

After being successfully authenticated service calls need to performed including the security token(s) in its soap headers. This method builds such a header.

The header of the given document will be enhanced so there is no need to process the returning value.

Parameters

doc:Request XmlMason document.
security_token:Security tokens as plain text gathered using the TokenService.

[Source]
